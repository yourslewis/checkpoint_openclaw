When people say *Google LFM*, they usually mean:
> A **large shared foundation model** trained on massive multi-task ads/retrieval data, fine-tuned using **logged feedback signals** (clicks, dwell, conversions) with **counterfactual correction**.

### Core Contributions
- **Unified Foundation Model**
  - One backbone (Transformer-based) serving:
    - query understanding
    - ad understanding
    - user/context embedding
  - Shared across Search Ads, Display, Shopping.

- **Learning From Feedback (LFM)**
  - Trained primarily on **implicit feedback**:
    - clicks, long-clicks
    - conversions
    - skips / dwell time
  - Uses **counterfactual learning** to correct for bias.

- **Offline + Online Loop**
  - Offline: logged bandit data
  - Online: controlled traffic, exploration buckets
  - Continuous refresh with fresh logs

### Technical Stack (High Level)
- Backbone: Transformer (encoder-style, not pure LLM)
- Objectives:
  - CTR prediction
  - CVR prediction
  - Ranking loss (pairwise / listwise)
- Feedback handling:
  - Inverse Propensity Scoring (IPS)
  - Self-normalized IPS
  - Doubly Robust estimators
- Scale:
  - Trillions of training examples
  - Sparse + dense features
